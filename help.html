Welcome to Assemblitz



What is Assemblitz

Assemblitz is a light web-based application that would quickly translate assembly instructions into machine code.


How to use Assemblitz


Translating Assembly Code to Machine Code

  1. Input assembly instructions in the text area provided found at the left side of website.
  2. Click on the "Assemble" button located at the bottom of the page to translate assembly instructions to its machine language equivalent.
  3. The equivalent machine language would be displayed at the upper right part of the page.

Viewing the Stack

  1. Click on the "View Stack" button located at the bottom of the web page to view the stack on a pop-up window.

Using the Assemblitz Instruction Set

  • read
    • Assembly code that would make the computer read an integer from the keyboard and store it to the variable name provided
    • Example usage: read number1
  • disp
    • Allows the user to display the value stored in the variable
    • Example usage: disp number1
  • pushi
    • Places the given 2-digit integer at the top of the stack register(SR), throws a Stack Overflow Error if the stack register is full
    • Example usage: push 28
  • pushv
    • Places the integer stored in a variable to the top of the stack register (SR) and throws a Stack Overflow Error if the stack register is full.
    • Example usage: push number1
  • pop
    • Removes the topmost integer found in the stack register and places it in the given variable. Throws an Empty Stack Error if the stack register is empty
    • Example usage: pop number2
  • mod
    • Performs modulo operation between the two topmost integers stored in the stack register and stores the result at the topmost location of the stack. If the second topmost location in the stack register is empty then it would throw a Null Operand Error.
    • Example usage: mod
  • jmp
    • Jumps to the specified memory location
    • Example usage: jmp add
  • jl
    • Jumps to the specified memory location if the value at the top of stack register is less than the value at the second-to-the top location in the stack register. If second-to-the-top location in the stack register is empty, then a Null Compare Error would be thrown.
    • Example usage: jl less
  • jg
    • Jumps to the specified memory location if the value at the top of stack register is greater than the value at the second-to-the top location in the stack register. If second-to-the-top location in the stack register is empty, then a Null Compare Error would be thrown.
    • Example usage: jg greater
  • jeq
    • Jumps to the specified memory location if the value at the top of stack register is equal to the value at the second-to-the top location in the stack register. If second-to-the-top location in the stack register is empty, then a Null Compare Error would be thrown.
    • Example usage: jeq equal
  • add
    • Removes the top-most and the second top-most integer stored in the Stack Register, adds the two values and then stores the sum at the top of the stack. If second-to-the-top location in the stack register is empty, the Null Operand Error would be thrown. Also if the sum is greater than 99, the Overflow Error would be thrown.
    • Example usage: add
  • sub
    • Removes the top-most and the second top-most integer stored in the Stack Register, subtracts the two values and then stores the difference at the top of the stack. If second-to-the-top location in the stack register is empty, the Null Operand Error would be thrown. Also if the difference is less than zero, the Overflow Error would be thrown.
    • Example usage: sub
  • cmp
    • Removes the top-most and the second top-most integer stored in the Stack Register, compares the two values and then stores the value 1 at the top of stack register if the two values are equal or zero if the two values are not.
    • Example usage: cmp
  • begin
    • Indicates the beginning of the assembly code and must be found at the first line of the code only. Otherwise the assembler would throw a Missing Begin Error.
    • Example usage: begin
  • begin
    • Indicates the end of the assembly code. All lines declared after the end instruction would be disregarded.
    • Example usage: end
